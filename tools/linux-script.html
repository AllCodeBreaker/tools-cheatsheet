Linux Cheatsheet - Scripting & Automation

  • #!/bin/bash: Shebang to specify script interpreter
  • chmod +x script.sh: Make script executable
  • Variables: VAR="value", accessed as $VAR
  • Control Structures:
    • if [ condition ]; then ... fi
    • for var in list; do ... done
    • while [ condition ]; do ... done
    • case $var in pattern) ... ;; esac
  • Comments: start lines with #
#!/bin/bash
for i in 1 2 3; do
  echo "Number $i"
done
CommandDescription
crontab -eEdit your user’s cron jobs
crontab -lList current cron jobs
sudo systemctl status cronCheck cron service status

Cron syntax: minute hour day month weekday command

# Run backup.sh every day at 2am
0 2 * * * /home/user/backup.sh

Alternative to cron using systemd timers for more advanced scheduling:

  • Create a .service file describing the job
  • Create a .timer file specifying schedule
  • Enable and start timer with:
sudo systemctl enable myjob.timer
sudo systemctl start myjob.timer

Example timer unit runs every day at midnight:

[Unit]
Description=Run myjob daily

[Timer]
OnCalendar=daily
Persistent=true

[Install]
WantedBy=timers.target

  • Check if file exists: 
    [ -f filename ] && echo "File exists"
  • Loop through files in directory: 
    for file in /path/*; do echo "$file"; done
  • Redirect output: 
    command > output.txt  # overwrite
command >> output.txt # append
  • Use pipes: 
    ls -l | grep ".log"
