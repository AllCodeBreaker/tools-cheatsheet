Nessus Cheatsheet

  • Nessus Server: Runs scans, manages plugins.
  • Client: Web interface or Nessus GUI to control scans.
  • Policies: Define scan configurations (credentials, plugins).
  • Plugins: Scripts that detect vulnerabilities.
  • Reports: Scan results with vulnerability details and severity.
Scan TypeDescription
Basic Network ScanChecks for open ports and common vulnerabilities.
Credentialed ScanUses credentials to perform in-depth host checks.
Web Application TestsDetects web vulnerabilities like XSS, SQLi.
Malware ScanChecks for malicious software indicators.
Compliance ScanValidates against compliance standards (PCI, HIPAA).
  • nessuscli update — Update plugins and Nessus engine.
  • nessuscli adduser <user> — Add user for the Nessus server.
  • nessuscli rmuser <user> — Remove user.
  • nessuscli start — Start Nessus service.
  • nessuscli stop — Stop Nessus service.
  • Use web UI at https://localhost:8834 for scan management.
  • Use credentialed scans for better accuracy.
  • Customize policies based on target environment.
  • Schedule scans during off-hours to reduce impact.
  • Review and tune plugins for performance.
  • Export reports in formats like PDF, CSV, or HTML.
  • Regularly update Nessus plugins for latest vulnerability detection.
  • Segment scans to avoid overwhelming networks.
  • Use multi-factor authentication on Nessus accounts.
  • Integrate Nessus with SIEM tools for centralized logging.
