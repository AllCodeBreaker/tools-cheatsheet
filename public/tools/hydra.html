Hydra Password Cracking Cheatsheet

hydra -L <userlist> -P <passlist> <target> <service>

Brute force login on a service using user and password lists.

ProtocolExample
SSHssh
FTPftp
HTTP-GEThttp-get
HTTP-POSThttp-post-form
SMBsmb
Telnettelnet
OptionDescription
-L <file>Usernames list
-l <user>Single username
-P <file>Password list
-p <password>Single password
-t <tasks>Number of parallel tasks (default 16)
-s <port>Port number if not default
-fExit after first valid password found
-vVVerbose mode, show each attempt
-o <file>Save output to file
hydra -L users.txt -P passwords.txt <target> http-post-form "/login.php:user=^USER^&pass=^PASS^:F=incorrect"

Brute force HTTP POST login form. Replace URL, parameters, and failure string.

  • Use appropriate wordlists like SecLists for usernames and passwords.
  • Adjust -t for speed vs stability.
  • Check target's login lockout policies to avoid bans.
  • Use verbose mode (-vV) for detailed feedback.
  • Combine with proxy tools to monitor traffic.
