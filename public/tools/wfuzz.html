Wfuzz Cheatsheet

  • Wfuzz is a flexible web application brute forcing tool.
  • Used for fuzzing URLs, parameters, headers, cookies, and POST data.
  • Supports wordlists and multiple injection points.
CommandDescription
wfuzz -c -w wordlist.txt --hc 404 http://target/FUZZFuzz URL path, hide 404 responses
wfuzz -c -w params.txt --hc 404 http://target/index.php?FUZZ=1Fuzz GET parameters
wfuzz -c -w wordlist.txt --hc 404 --hs "Error" http://target/FUZZHide responses containing "Error"
wfuzz -c -w wordlist.txt -d "username=admin&password=FUZZ" --hc 302 http://target/login.phpFuzz POST parameter
wfuzz -c -w wordlist.txt -H "Cookie: session=FUZZ" http://target/Fuzz cookie values
  • -c: Colored output
  • -w <wordlist>: Specify wordlist file
  • --hc <codes>: Hide HTTP response codes (e.g. 404)
  • --hs <string>: Hide responses containing string
  • --sc <codes>: Show only specific HTTP response codes
  • -d <data>: POST data for fuzzing
  • -H <header>: Add HTTP headers
  • Use appropriate wordlists targeting your application.
  • Combine filters (--hc, --hs, --sc) to reduce noise.
  • Test different injection points (URL, headers, POST).
  • Use verbose and colored output for easier analysis.
  • Respect target and avoid unnecessary flooding.
