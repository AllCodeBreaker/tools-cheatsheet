Amass Complete Cheatsheet

amass <command> [flags]

Example:
amass enum -d example.com
CommandDescription
enumPerform subdomain enumeration
vizVisualize data (e.g., in Maltego)
trackTrack changes between enumerations
intelDiscover related domains and organizations
dbQuery the local Amass database
serverStart a local data collection server
-d <domain> — Target domain (required)
-df <file> — File with list of domains
-oA <name> — Output in all formats (TXT, JSON, Graphviz)
-o <file> — Output results to a file
-src — Show source of each result
-ip — Include IP addresses
-brute — Enable brute-force mode
-w <wordlist> — Use custom wordlist
-active — Enable active recon (e.g., DNS resolution)
-passive — Use only passive data sources
-r <resolver> — Specify DNS resolver
-timeout <sec> — Set timeout
-max-dns-queries <int> — Limit DNS requests
-oA name — Generates:
  • name.txt — Plain text list
  • name.json — Structured JSON output
  • name.graphml — Graph visualization format
Config file: ~/.config/amass/config.ini

Add API keys for sources like:
  • Shodan
  • VirusTotal
  • SecurityTrails
  • Spyse
  • WHOISXML
Example:
[Shodan]
apikey = YOUR_API_KEY

[SecurityTrails]
apikey = YOUR_API_KEY
🔍 **Passive scan**:
amass enum -passive -d example.com

🚀 **Active brute-force with custom wordlist**:
amass enum -brute -w words.txt -d example.com

📦 **Save output in all formats**:
amass enum -d example.com -oA results/example

📚 **Enumerate from file of domains**:
amass enum -df domains.txt -o results.txt
