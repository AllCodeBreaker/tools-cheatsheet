Classes and Interfaces

Servlet (Interface)

Purpose / Use: Defines the lifecycle methods a servlet must implement. Base interface for all servlets.

Constructors

Interfaces do not have constructors.

Public Methods


void init(ServletConfig config) throws ServletException
void service(ServletRequest req, ServletResponse res) throws ServletException, IOException
void destroy()
ServletConfig getServletConfig()
String getServletInfo()

Description:

  • init(ServletConfig): Initializes the servlet once at startup.
  • service(ServletRequest, ServletResponse): Handles client requests.
  • destroy(): Called before the servlet is removed from service.
  • getServletConfig(): Returns the servlet configuration object.
  • getServletInfo(): Returns servlet information like version or author.

GenericServlet (Abstract Class)

Purpose / Use: Abstract base class implementing the Servlet interface with default lifecycle methods except for service(), which is abstract.

Constructors


GenericServlet()

Default constructor initializes the servlet.

Public Methods

abstract void service(ServletRequest req, ServletResponse res) throws ServletException, IOException

Description: Provides default implementations for init, getServletConfig, getServletInfo, and destroy. Subclasses must implement service() for request handling.
