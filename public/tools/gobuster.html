Gobuster Complete Cheatsheet

gobuster dir -u http://example.com -w /path/to/wordlist.txt

Directory brute forcing on target URL with specified wordlist.

ModeDescription
dirDirectory/File brute forcing
dnsDNS subdomain brute forcing
vhostVirtual host brute forcing
fuzzFuzzing using custom patterns
FlagDescription
-u <url>Target URL (for dir/vhost/fuzz)
-w <file>Wordlist file path
-t <threads>Number of concurrent threads (default: 10)
-x <extensions>File extensions to append (e.g. php,html)
-o <file>Save output to file
-kIgnore SSL certificate errors
-s <status codes>Show only responses with these HTTP status codes (e.g. 200,204,301)
  • Brute force directories:
    gobuster dir -u http://example.com -w /usr/share/wordlists/dirbuster/directory-list-2.3-medium.txt
  • Brute force subdomains:
    gobuster dns -d example.com -w subdomains.txt
  • Brute force vhosts:
    gobuster vhost -u http://example.com -w vhosts.txt
  • Scan with multiple extensions:
    gobuster dir -u http://example.com -w wordlist.txt -x php,html,txt
  • Use realistic and updated wordlists for better results.
  • Combine Gobuster with other tools like Nikto and Burp Suite.
  • Adjust thread count for speed/performance tradeoff.
  • Filter results using status codes to remove noise.
